# Pizza Sales Project
The Pizza Sales Data Analysis project is designed to analyze pizza sales data, providing actionable insights to improve sales performance and business operations. The project includes data import, SQL querying, data cleaning, and visualization using Microsoft tools such as MS SQL Server and Power BI. The objective is to create a comprehensive analysis pipeline that helps a pizza business optimize sales strategies, predict customer behavior, and generate insightful reports.
## Requirement:

We need to analyze key indicators for our pizza sales data to gain insights into our business performance. Specifically, we want to calculate the following metrics.

- Total Revenue: The sum of the total price of all pizza orders.
-  Average Order Value: The average amount spent per order, calculated by dividing the total revenue by the number of orders.
- Total Pizzas Sold: The sum of the quantities of all pizzas sold
-  Total Orders: The total number of orders placed
-  Average Pizzas Per Order: The average number of pizzas sold per order, calculated by dividing the total number of pizzas sold by the total number of orders.

## Dataset Description:
The dataset used includes typical pizza sales transaction data, such as:

Order ID: Unique identifier for each order.
Date/Time: The date and time of each order.
Pizza Type: The type of pizza ordered (e.g., Margherita, Pepperoni).
Quantity: The number of pizzas ordered.
Price: The price of each pizza.
Total Sale Amount: The total revenue generated by each order.
Customer Information: Customer ID, location, and delivery method, if applicable.

## Objective
The main objectives of this extended project are:

Import and manage raw sales data in a relational database (MS SQL Server).
Use SQL queries to extract meaningful insights from the pizza sales data.
Build data reports and interactive dashboards using Power BI to visualize sales trends.
Perform data cleaning and transformation using Power Query.
Leverage DAX (Data Analysis Expressions) for data processing and more advanced calculations within Power BI.

## The project incorporates the following functionalities and processes:

Database Creation and Data Import: The raw pizza sales data is imported into MS SQL Server, a relational database, where it is organized for efficient querying and processing.
SQL Querying: SQL queries are written to extract key insights, such as total sales per pizza type, daily sales trends, and the most profitable items.
Data Cleaning and Processing: Using Power Query in Power BI, the data undergoes further cleaning, including handling missing values, transforming data types, and creating new calculated columns for better analysis.
Data Visualization: The processed data is visualized in Power BI through interactive charts, dashboards, and reports, highlighting the key performance indicators (KPIs) for the pizza business.
Dashboard Creation: A Pizza Sales Dashboard is generated to provide a holistic view of the business’s sales performance, enabling quick decision-making.

## Strategy Used
1. A relational database is created in MS SQL Server to store the pizza sales data.
- The raw sales data is imported into the database using SQL Server Import Wizard.
2. SQL queries are written to analyze key aspects of the pizza sales, including:
- Total sales by pizza type.
- Daily and weekly sales trends.
- Best-selling pizzas and least popular items.
- Customer order frequency and revenue analysis.
- Queries are executed within MS SQL Server to generate reports from the raw data.
3. Create Reports from SQL Server:
- Reports are generated by executing SQL queries directly from MS SQL Server, which helps summarize key findings such as total revenue, sales distribution across products, and time-based trends.
  
4. Connect MS SQL Server to Power BI:

- Power BI is connected to the MS SQL Server database to fetch the raw or processed data for further analysis.
- This connection allows for seamless data updates and visualization directly from the database.
5. Data Cleaning using Power Query:

- Using Power Query, the data is cleaned and transformed to ensure it is suitable for analysis. This includes:
Removing duplicates.
Handling missing or null values.
Changing data types (e.g., converting date strings to datetime objects).
Creating new features such as total sales per order or time-based features (day, month, year).

6. Data Processing using DAX:

- DAX (Data Analysis Expressions) is used within Power BI to create complex calculations and aggregations, such as:
Calculating average sales per pizza type.
Total revenue by week or month.
Creating custom metrics like customer lifetime value or sales per customer segment.
- DAX helps derive deeper insights from the data, allowing for more advanced analytics in the Power BI reports.
7. Data Visualization:

- Using Power BI, the data is visualized through:
   - Bar charts and Pie charts showing sales distribution by pizza type.
   - Line charts tracking daily, weekly, and monthly sales trends.
   - Heatmaps highlighting peak sales periods (time of day, day of the week).
   - KPI cards displaying key metrics like total revenue, number of orders, and average sales per customer.
8. Generating Pizza Sales Reports and Dashboards:

- Power BI Dashboards are created, showcasing:
    - Sales performance over time (daily/weekly/monthly).
    - Most popular pizzas and best-selling categories.
    - Revenue distribution by customer segment or location.
    - Overall business performance through high-level KPIs.
- Reports and dashboards are interactive, allowing business owners to filter data and drill down into specific insights.

## 5. Conclusions with Findings
- Peak Sales Times and Pizza Preferences:
Analysis of the data likely shows that certain days of the week (e.g., weekends) and times of the day (e.g., dinner hours) experience higher sales. Popular pizza types, such as Pepperoni or Margherita, contribute significantly to total revenue, while specialty pizzas may show variable performance.

- Operational Efficiency:
By identifying peak demand periods and product popularity, the business can optimize its inventory and staffing schedules, reducing waste during off-peak times and ensuring high demand items are always available.

- Customer Insights and Preferences:
Through segmentation analysis (using DAX calculations), the project likely highlights customer preferences based on demographic or location data. For instance, certain customer segments might prefer specialty pizzas, or some locations might generate higher orders during specific periods.

- Model Accuracy and Improvements:
While this project is more focused on reporting and analysis than predictive modeling, improvements in data visualization and DAX formulas are likely to help refine business insights. Advanced DAX calculations, such as dynamic date filtering and custom aggregation functions, could improve the precision of key performance indicators (KPIs).

## 6. Use Case in Real-World Scenarios
- Business Decision-Making:
The dashboards created through Power BI allow pizza business owners and managers to make data-driven decisions by identifying which products to promote, when to scale up operations, and where to allocate resources for maximum profit.

- Inventory and Staffing Optimization:
By understanding peak demand times and best-selling products, businesses can optimize inventory management (minimizing waste) and staffing levels (ensuring adequate coverage during peak hours while reducing labor costs in slower periods).

- Marketing and Promotions:
Insights from the analysis can inform targeted marketing strategies. For example, the business could run promotions on less popular pizza types during peak hours or introduce combo deals for best-selling pizzas to further boost sales.

- Customer Retention and Growth:
The ability to analyze customer behavior over time allows the business to identify loyal customers and tailor promotions or loyalty programs to retain high-value customers. Additionally, segmentation analysis helps in understanding different customer groups and tailoring offerings to meet their preferences.
